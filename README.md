+ 区分v-if和v-show
  + v-if：适用于不会重复变更的情况下，因为它每次都会销毁重建
  + v-show：适用于重复变更的情况下，因为它只是显示隐藏
+ 能用computed就不要用watch
  + computed 本身是惰性的，并且具有缓存
+ 避免同时使用v-for和v-if
  + v-for的优先级高于v-if
+ 路由懒加载
  + 提高首屏渲染速度
+ 销毁自定义的事件
  + 要在beforeDestroy中销毁事件
+ 降低watcher
  + 减少data中的属性数量
  + 使用Object.freeze 冻结对象
+ 列表中一定使用key
  + 实际项目中列表都很复杂
  + 避免原地复用，快速找到对比的dom节点
+ 展示组件使用函数式组件
  + 本身没有data，不存在data相关的watcher
  + 本身是函数，不存在实例
+ 使用keep-alive
  + 将经常切换显示的组件缓存
  + 避免多次销毁创建的性能损失
